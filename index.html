Lenguaje de marcas

Un lenguaje de marcado o lenguaje de marcas es una forma de codificar un documento que, junto con el texto, incorpora etiquetas o marcas que contienen información adicional acerca de la estructura del texto o su presentación. Los Lenguajes de Marcas permiten hacer explícita la estructura de un documento, su contenido semántico o cualquier otra información lingüística o extralingüística que se quiera hacer patente.

Sistemas de gestión de información

Un Sistema de gestión de información son un tipo de sistemas de informació que recopilan y procesan información de diferentes fuentes. Esta información sirve para crear informes que proporcionan una visión completa de toda la información necesaria para tomar decisiones que van desde pequeños detalles diarios hasta una estrategia de nivel superior. Los sistemas actuales de gestión de la información se basan en gran medida en la tecnología para recopilar y presentar datos.

ERP (sistema de gestión empresarial)

Conjunto de sistemas de información que permite la integración de ciertas operaciones de una empresa, especialmente las que tienen que ver con la producción, la logística, el inventario, los envíos y la contabilidad.

Funciona como un sistema integrado, un único programa con acceso a una base de datos centralizada.

El propósito de un software ERP es apoyar a los clientes de la empresa, dar tiempos rápidos de respuesta a sus problemas, así como un eficiente manejo de información que permita la toma de decisiones y minimizar los costes.

Caracteristicas de los siguientes lenguajes de marcas:

HTML

La sigla corresponde a HyperText Markup Language, es decir, Lenguaje de Marcas de Hipertexto.

Se encarga de desarrollar una descripción sobre los contenidos que aparecen como textos y sobre su estructura, complementando dicho texto con diversos objetos (como fotografías, animaciones, etc).

Puede ser creado y editado con cualquier editor básico de textos.

Es multiplataforma, puede ser visualizado por cualquier navegador de cualquier sistema operativo.

No diferencia entre mayúsculas y minúsculas.

Utiliza etiquetas o marcas, breves instrucciones de comienzo y final, que determinan la forma en la que debe aparecer en su navegador el texto así como las imágenes y demás elementos en la pantalla del ordenador.

Cada elemento de un documento HTML consta de una etiqueta de comienzo, un bloque de texto y una etiqueta de fin. Ej:< y >.

Lenguaje estático utilizado para la creación de páginas web.

Los documentos HTML son documentos de hipertexto que aparecen enlazando a otros documentos.

Es un estándar reconocido por todo el mundo y cuyas normas define un organismo sin animo de lucro llamado World Wide Web Consortium, más conocido como W3C.

Cabe destacar que el HTML permite ciertos códigos que se conocen como scripts, los cuales brindan instrucciones específicas a los navegadores que se encargan de procesar el lenguaje. Entre los scripts que pueden agregarse, los más conocidos y utilizados son JavaScript y PHP.

CSS

Un lenguaje de hojas de estilo que permite a desarrolladores y usuarios asociar formas de presentación a documentos estructurados, como páginas HTML y aplicaciones XML

Complementariedad con documentos estructurados

Complementan a otros documentos proporcionando información de estilo con poco o ningún impacto sobre el sistema de marcas.

Independencia del vendedor, la plataforma y el dispositivo

Mantenibilidad

Los responsables de sitios en la Web pueden simplificar el mantenimiento y conservar un estilo y un efecto consistente a todo lo largo del sitio.

Simplicidad

Lenguaje de estilo simple, humanamente legible y posible de escribir.
Generalmente sólo hay un modo de conseguir un efecto determinado

Rendimiento de la red

Proporciona una compacta codificación para presentar los contenidos.
Generalmente disminuyen el tamaño del contenido

Flexibilidad

Pueden ser aplicadas de varias maneras.
Característica clave: su capacidad de formar una cascada de estilos con los distintos grupos de información

Riqueza

Aumentan la riqueza de la Web como medio de expresión con sus abundantes efectos de procesamiento
Alta Funcionalidad en programas de edición y de presentaciones gráficas.

Combinación con lenguajes alternativos

Accesibilidad



XHTML

Los documentos XHTML son conformes a XML. Como tales, son fácilmente visualizados, editados y validados con herramientas XML estándar.

Los documentos XHTML pueden escribirse para que funcionen igual o mejor que lo hacían antes tanto en las aplicaciones de usuario conformes a HTML 4.0 como en los nuevas aplicaciones conformes a XHTML 1.0.

Los documentos XHTML pueden usar aplicaciones (por ejemplo scripts y applets) que se basen ya sea en el Modelo del Objeto Documento de HTML o XML[DOM].

A medida que la familia XHTML evolucione, los documentos conformes a XHTML 1.0 estarán más preparados para interactuar dentro de y entre distintos entornos XHTML.


XML

XML son las siglas de Extensible Markup Language, que podemos traducir como Lenguaje de Marcas Extensibles, aunque realmente es un meta-lenguaje.

Se compone de etiquetas que nos aportan datos e información que queremos procesar. Estas etiquetas pueden estar de forma individual o anidadas.

XML es un estándar para escribir datos estructurados en un fichero de texto.

Tanto XML como HTML usan marcas y atributos, más su diferencia radica en que en XML sólo se usan las marcas para delimitar fragmentos de datos, dejando la interpretación de éstos a la aplicación que los lee.

Puede ser usado en cualquier plataforma destaca por su portabilidad, depuración, independencia de plataforma, e incluso en su capacidad de edición, pero su sintaxis es más estricta que la de HTML

XML consta de una familia de tecnologías

Los ficheros resultantes, son casi siempre mayores que sus equivalentes binarios

XML es nuevo empezó a diseñarse en 1996, y se publicó la recomendación en Febrero/98

XML no requiere licencia


MARKDOWN

Markdown es un lenguaje de marcado ligero creado por John Gruber que trata de conseguir la máxima legibilidad y facilidad de publicación tanto en su forma de entrada como de salida, inspirándose en muchas convenciones existentes para marcar mensajes de correo electrónico usando texto plano

Se distribuye bajo licencia BSD como plugin en diferentes sistemas de gestión de contenidos (CMS)

Convierte el texto marcado en documentos XHTML utilizando html2text

Pensado para elaborar textos cuyo destino iba a ser la web con más rapidez y sencillez que si estuviésemos empleando directamente HTML

No hay un estándar definido para Markdown, aparte de la implementación original de John Gruber, que algunos consideran obsoleto. Esto está ocasionado por la fragmentación, porque las diferentes personas escriben sus propias variantes de Markdown para corregir detalles o añadir características no incluidas.


XPATH

Xpath (XML Path Language) es un lenguaje que permite recuperar información de un documento XML. Permite construir expresiones que recorren y procesan el documento XML.

Utiliza un tipo de notación similar a las rutas de los ficheros, pero haciendo referencia a los nodos de un XML.

fue creado para su uso con el estándar XSTL (es el mayor componente en XSTL).

Define una sintaxis para establecer partes en un documento XML, permitiendo navegar a través de sus elementos y atributos y editar algunos de éstos.


XQUERY

Es un lenguaje de consulta que permite extraer información de bases de datos o documentos XML

XQuery es a XML lo mismo que SQL a las bases de datos relacionales.

Se basa en el lenguaje Xpath para el acceso a los nodos XML, pudiendo utilizar todos sus operadores y funciones.

Se trata de una recomendación del W3C

El lenguaje XQuery es muy amplio y complejo consultas XQuery se componen de cinco cláusulas, que debido a sus iniciales se las conoce como FLWOR
(For, Let, Where, Order by, Return. Además se le añade la cláusula Group By)
